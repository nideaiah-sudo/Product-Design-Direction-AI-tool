# standard-parts-ai 更新日志

> 本次更新较上一版本新增/改进功能总览

## 1. PDF 与照片识别路径彻底分离

- 新增 `pdf_parser.py`（原 `parsers.py`），只处理 PDF 文字、工程图公差、文件名解析；
- `image_parser.py` 完全独立，维护自己的 OCR 清洗、正则、尺寸兜底逻辑；
- 两个路径不再交叉 import，互不干扰。

## 2. GUI 新增参数手动修正

- 新增“参数修正”区域，可编辑：螺纹大径、螺距、螺杆长度、头部直径、头部高度、六角对边/槽号；
- 参数框默认只读，点击 **“调整”** 按钮后解锁；
- 新增 **“按标准重置”** 按钮，一键按 ISO 标准恢复头径、头高、螺距；
- 生成 STEP 时优先使用手动修正后的参数。

## 3. 照片 OCR 识别准确率提升

- 多 `M<大径>` 候选择优：OCR 同时出现 M5/M12/M1.2 等噪声时，根据长度、头径、螺距打分选出最合理的大径；
- 螺距-大径一致性校验：识别到 0.5mm 螺距时自动对应 M3，避免把 M3×10 误认成 M5；
- 长度兜底：标题 `M4*6` 被 OCR 误读成 `M4*33` 时，用图中真实 `6mm` 修正；
- 头径/头高兜底：避免把长度误当头部直径；
- 新增 `M1.2` 标准尺寸支持。

## 4. 标准件库扩展

- `standards.py` 增加 M1.2 尺寸、螺距、键名映射；
- 头部尺寸表覆盖 M1.2 ~ M10。

## 5. 打包与验证

- 已重新打包 `dist\standard-parts-ai\standard-parts-ai.exe`；
- 单元测试 28 个全部通过。
